Arsitektur Skema Master Data NusantaraTrip SaaS1. Apakah Master Data Memberatkan API / Database / Response?Jawabannya: TIDAK, justru sebaliknya. Master data yang dipisah (dijadikan tabel referensi) akan membuat database jauh lebih ringan, konsisten, dan terhindar dari duplikasi data (redundansi).Namun, agar Response API sangat cepat (0.1 detik), Backend harus menerapkan Caching Strategy:Redis / Memcached: Karena Master Data (seperti daftar Destinasi atau Kategori) jarang berubah, Backend tidak boleh melakukan SELECT * FROM destinations setiap kali user membuka homepage. Data ini harus di-simpan di RAM Cache (Redis).State Management (Frontend): Di sisi Frontend (React/Vue), data master dipanggil sekali saat aplikasi load, lalu disimpan di State (Redux/Zustand) atau localStorage. Sehingga saat user pindah dari halaman "Home" ke halaman "Filter Pencarian", Frontend tidak perlu memanggil API lagi.2. Struktur Skema Database Master Data (Blueprint)Daripada membuat 1 tabel raksasa, praktik terbaik (Best Practice) adalah membaginya berdasarkan fungsi operasional.A. Tabel master_destinations (Destinasi Wisata)Tabel ini sangat krusial karena akan digunakan untuk men-generate halaman Explore Destination, SEO Pages (Contoh: /destination/bali), dan Filter Pencarian (Grid Card).Kolom / Field yang wajib ada:id (Primary Key, UUID/Integer)name (String) -> Contoh: "Bali", "Gunung Bromo"slug (String, Unique) -> Contoh: bali, gunung-bromo (Sangat penting untuk URL SEO Frontend).thumbnail_url (String) -> Gambar kotak/portrait untuk Grid Card pencarian.cover_image_url (String) -> Gambar resolusi tinggi memanjang (Hero Image) saat halaman Explore Destination dibuka.short_description (Text) -> 2 kalimat untuk di Card Grid.full_description (Long Text / HTML) -> Artikel lengkap dari CMS untuk halaman Explore.region (String) -> Contoh: "Jawa Timur", "Bali & Nusa Tenggara" (Untuk filter Dropdown).is_featured (Boolean) -> Jika true, otomatis muncul di Homepage bagian "Destinasi Populer".status (Enum: 'active', 'draft')B. Tabel master_categories & master_trip_types (Kategori & Tipe Trip)Digunakan sebagai badge di pojok gambar paket tour, ikon di halaman pencarian, dan filter di sidebar.Kolom / Field yang wajib ada:id (Primary Key)name (String) -> Contoh: "Honeymoon", "Open Trip", "Family Gathering"slug (String, Unique) -> Contoh: honeymoon, open-tripicon_code (String) -> Class font-awesome atau URL SVG. Contoh: fa-solid fa-heart atau /icons/honeymoon.svg.description (Text)status (Enum: 'active', 'inactive')C. Tabel master_addons (Fasilitas Tambahan Berbayar)Digunakan di Smart Tour Builder dan halaman Checkout B2C (saat user mau nambah fasilitas).Kolom / Field yang wajib ada:id (Primary Key)name (String) -> Contoh: "Sewa Jeep 4WD", "Dokumentasi Drone"pricing_type (Enum) -> per_pax (harga x jumlah orang) atau per_group (harga flat/pukul rata). Sangat Penting untuk kalkulasi kalkulator harga!default_price (Decimal/Integer) -> Harga dasar (Misal: 600000). Saat Admin bikin paket, harga ini otomatis muncul, tapi Admin bisa override (mengubahnya khusus untuk paket tersebut).context_destination_id (Foreign Key, Opsional) -> Jika diisi, Add-on "Jeep Bromo" HANYA muncul jika Admin membuat paket dengan destinasi Bromo.status (Enum)D. Tabel master_locations (Titik Penjemputan / Transportasi)Digunakan di Dropdown Meeting Point pada halaman Detail Paket, atau sebagai master data rute.Kolom / Field yang wajib ada:id (Primary Key)name (String) -> Contoh: "Bandara Juanda", "Stasiun Malang Kota Baru"type (Enum) -> airport, train_station, hotel, public_space. (Berguna untuk menampilkan icon yang berbeda di UI).latitude (Decimal) & longitude (Decimal) -> Untuk digambar otomatis di Google Maps Frontend.status (Enum)E. Tabel master_inclusions_exclusions (Include / Exclude)Digunakan di Smart Tour Builder (Checkbox list) dan di Halaman Detail Paket (Tab Fasilitas dengan ikon centang hijau / silang merah).Kolom / Field yang wajib ada:id (Primary Key)name (String) -> Contoh: "Tiket Masuk Wisata", "Makan Siang", "Tips Guide"default_type (Enum) -> include atau exclude. (Sebagai saran otomatis saat admin membuat paket).icon_code (String) -> Contoh: fa-ticket, fa-utensils.3. Relasi Database (Cara Menyambungkan Master Data ke Paket Tour)Saat Admin menggunakan Smart Tour Builder (menyimpan data paket tour), struktur relasi database-nya menggunakan Pivot Tables (Many-to-Many).Tabel tours akan memiliki relasi ke:tour_destinations (Menyambungkan 1 paket ke banyak destinasi, misal Paket Bali menyentuh Kuta dan Ubud).tour_addons (Menyambungkan Addons, di sini admin bisa memasukkan custom_price jika harga Jeep di paket A berbeda dengan paket B).tour_inclusions (List apa saja yang didapat).tour_itineraries (Tabel anak dari Tour, berisi day, start_time, title).4. Saran & Tips (Developer Handoff) untuk Frontend-Backend IntegrationBeri tahu Backend Developer Anda 4 aturan emas ini untuk mempermudah integrasi dengan UI yang telah kita buat:Wajib Gunakan "Soft Deletes":Jangan pernah menggunakan perintah DELETE FROM master_addons. Gunakan kolom deleted_at. Mengapa? Jika Add-on "Sewa Drone" dihapus permanen, maka Invoice/Riwayat Pesanan pelanggan tahun lalu yang menggunakan Drone akan error/hancur saat dicetak PDF-nya.Standardisasi JSON Response:Sepakati bentuk kembalian API. Format terbaik yang sering dipakai di ekosistem React/Vue adalah standar JSON:API.{
  "success": true,
  "message": "Destinations fetched successfully",
  "data": [ ... ],
  "meta": {
     "current_page": 1,
     "last_page": 5,
     "total": 45
  }
}
Endpoint Bulk & Dropdown Khusus:Buat endpoint API yang ringan khusus untuk Select Dropdown di Admin. Jangan mengirim deskripsi panjang dan URL gambar jika frontend hanya butuh ID dan Nama untuk mengisi tag <select>.Contoh Endpoint: GET /api/v1/admin/master-data/destinations?format=dropdown (hanya me-return id dan name).Fitur Duplicate / Clone Tour:Mengingat Smart Tour Builder sangat kompleks (ada Itinerary, Addons, Inclusions), Backend wajib membuat 1 endpoint POST /api/v1/tours/{id}/clone. Jadi Admin bisa menduplikasi Paket Bromo Reguler menjadi Paket Bromo VIP dengan 1 klik, lalu tinggal mengubah hotelnya, tanpa harus mengetik Itinerary dari nol.
