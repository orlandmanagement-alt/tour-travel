# Task & API Integration Analysis (2026-03-31)

## 1. Database Schema Status (`schema.md` vs `apps/api/src/db/schema.sql`)
✅ **Status Database:** Tersinkronisasi dengan baik dan siap.

Terdapat 16 tabel yang sudah di-generate termasuk relasi-relasinya (semua 14 tabel dari `schema.md` sudah tercakup, plus ada penambahan logis seperti `tour_reviews` dan `master_addons` untuk pelengkapan fitur).
- **Tindakan Lanjutan Database:** Tidak ada modifikasi tabel atau kolom yang perlu dibuat saat ini; skema sudah memadai untuk workflow saat ini.

## 2. Analisis Frontend - Web App (Public)
Berikut adalah daftar page/komponen yang **belum terintegrasi** dengan API backend dan masih sebatas UI/UX atau menggunakan Mock Data:
- ❌ **Homepage** (`apps/web/src/app/page.tsx`): Menampilkan list paket Best Sellers, Categories, dan Promos namun sumber datanya masih di-*hardcode* (`mock const`). Harus dibuat dinamis dengan GET ke Tour Controller/Master Controller.
- ❌ **Custom Trip Submission** (`apps/web/src/app/custom-trip/page.tsx`): Form *wizard* telah selesai, namun fungsi `handleSubmit` dan *loading state* masih berupa `setTimeout` buatan. Perlu trigger POST ke endpoint `CustomTripController`.
- ❌ **Promos & Destinations View** (`apps/web/src/app/promo/page.tsx`, `destinations/page.tsx`): Secara visual hanya UI tanpa pemanggilan data dari API.
- ❌ **Komentar/Ulasan** (`apps/web/src/components/ReviewSection.tsx`): Rata-rata review dan komentar user belum bersumber dari data di DB.

✅ *Bagian yang sudah tersinkron dan terkoneksi ke backend:*
- Tours Catalog & Detail (`/tours` dan `/tours/[id]`)
- Proses Checkout (`/checkout/[tourId]`)
- Sistem Tiket (`/ticket/[bookingRef]`)
- Authentikasi (Login/Register/Profile)

## 3. Analisis Frontend - Admin Dashboard
Berikut adalah daftar halaman dashboard yang **belum terintegrasi** dan hanya berupa layout UI/UX:
- ❌ **Master Data Manager** (`apps/admin/src/app/master/page.tsx`): Tabel CRUD (Lokasi, Tipe Trip) yang dibuat masih bersumber dari state lokal `MOCK_DATA`. Perlu endpoint GET/POST/PUT/DELETE dihubungkan ke `MasterController`.
- ❌ **Bookings List** (`apps/admin/src/app/bookings/page.tsx`): List transaksi pemesanan pada Admin mengandalkan array `mockBookings`. Perlu pull data real yang terhubung ke `BookingController`.
- ❌ **Manajemen Request Custom Trips** (`apps/admin/src/app/custom-trips/page.tsx`): Modul ini belum menangkap dan menampilkan data custom trip yang dikirim customer secara live.
- ❌ **Page Laporan & Metrik lainnya**: `analytics`, `blog`, `promos`, `settings/payments`, dan `users` semuanya masih berupa representasi layout dan belum memiliki fungsional logic.

✅ *Bagian yang sudah tersinkron dan terkoneksi ke API di sisi admin:*
- Auth Gateway Dashboard (`/login`)
- Pengaturan Paket Liburan Baru (`/tours/new` & `/tours/bulk-import`)
