🚀 API Integration Blueprint & ChecklistProject: NusantaraTrip Enterprise SaaS Travel PlatformDocument Type: Backend & Frontend Integration Handoff🏗️ Aturan Global Arsitektur API (Wajib)Sebelum masuk ke daftar endpoint, pastikan Backend mengikuti standar berikut:Format Response Wajib: JSON (application/json).Autentikasi: Menggunakan Token (JWT / Laravel Sanctum).Pagination: Setiap daftar (seperti daftar pesanan, daftar log) wajib menggunakan pagination (misal: 15 item per halaman) untuk mencegah database query yang berat.Soft Deletes: Jangan gunakan Hard Delete untuk Master Data dan Transaksi.CORS: Atur agar API bisa menerima request dari domain Frontend.📋 DAFTAR ENDPOINT YANG HARUS DIBANGUN🔐 MODULE 1: Authentication & User ManagementTerhubung dengan: auth-login.html, profile-settings.html, admin-users.html[ ] POST /api/auth/register : Mendaftarkan akun baru (B2C/B2B).[ ] POST /api/auth/login : Login email & password (mengembalikan Token).[ ] POST /api/auth/sso/{provider} : Login via Google/Apple.[ ] POST /api/auth/otp/send : Memanggil API WhatsApp pihak ketiga untuk mengirim 6-digit OTP.[ ] POST /api/auth/otp/verify : Memvalidasi OTP dan mengaktifkan akun.[ ] GET /api/user/profile : Mengambil data pengguna yang sedang login (termasuk saldo NusaPoin).[ ] PUT /api/user/profile : Update profil dan ganti password.[ ] GET /api/admin/users : (Admin) Mengambil daftar seluruh pengguna dengan filter (Reguler, Afiliasi, Corporate).[ ] PUT /api/admin/users/{id}/suspend : (Admin) Memblokir akses pengguna.🗺️ MODULE 2: Master Data ManagementTerhubung dengan: admin-master-data.html(Setiap entitas seperti Locations, Add-ons, Categories, Destinations membutuhkan 5 rute standar ini)[ ] GET /api/master/{entity} : List semua data (Admin & Publik).[ ] GET /api/master/{entity}/dropdown : List ringan (hanya ID dan Nama) untuk mengisi <select> di form pencarian Frontend.[ ] POST /api/master/{entity} : (Admin) Tambah data baru.[ ] GET /api/master/{entity}/{id} : (Admin) Detail 1 data untuk form Edit.[ ] PUT /api/master/{entity}/{id} : (Admin) Update data.[ ] DELETE /api/master/{entity}/{id} : (Admin) Hapus data (Soft Delete).[ ] POST /api/master/{entity}/bulk-import : (Admin) Upload CSV untuk import massal.📦 MODULE 3: Product Engine (Smart Tour Builder & Cars)Terhubung dengan: admin-tour-builder.html, admin-tour-crud.html, tour-list.html, tour-detail.html[ ] GET /api/tours : (Publik) Katalog paket wisata (mendukung filter lokasi, kategori, harga, durasi).[ ] GET /api/tours/{slug} : (Publik) Detail lengkap paket (termasuk join tabel itinerary, inclusions, dan addons).[ ] POST /api/admin/tours : (Admin) Endpoint Kompleks. Menerima Nested JSON dari Smart Builder (Metadata + Array Itinerary + Array Addons).[ ] PUT /api/admin/tours/{id} : (Admin) Update paket.[ ] POST /api/admin/tours/{id}/clone : (Admin) Menduplikasi paket agar admin tidak perlu membuat dari nol.🛒 MODULE 4: Booking & Checkout Engine (Core)Terhubung dengan: checkout.html, payment.html, admin-orders.html[ ] POST /api/checkout/calculate : Menerima input (ID Paket, Tgl Berangkat, Jumlah Pax, ID Add-ons terpilih, Kode Promo) -> Mengembalikan Rincian Harga (Subtotal, PPN, Diskon, Grand Total).[ ] POST /api/checkout/create : Membuat pesanan baru, memasukkan data Passenger Manifest, mengurangi kuota paket (Allotment), dan men-generate Invoice ID.[ ] POST /api/payment/webhook : SANGAT PENTING. Endpoint ini tidak dipanggil oleh Frontend, melainkan oleh Midtrans/Xendit saat tamu selesai transfer. Mengubah status order menjadi PAID.[ ] GET /api/orders/me : (User) Riwayat pesanan akun tersebut.[ ] GET /api/admin/orders : (Admin) List seluruh pesanan.[ ] POST /api/admin/orders/{id}/refund : (Admin) Memproses status refund.🤝 MODULE 5: Affiliate & B2B SystemTerhubung dengan: affiliate.html, user-affiliate.html, admin-affiliate-detail.html[ ] POST /api/affiliate/register : Mendaftar sebagai mitra.[ ] GET /api/affiliate/dashboard : Mengambil metrik (Total Klik, Estimasi Komisi, Saldo Tersedia).[ ] GET /api/affiliate/commissions : Riwayat komisi masuk dari transaksi orang lain yang menggunakan kode mereka.[ ] POST /api/affiliate/payout/request : (Mitra) Meminta pencairan dana ke rekening bank.[ ] GET /api/admin/payouts : (Admin) Daftar permintaan pencairan.[ ] POST /api/admin/payouts/{id}/approve : (Admin) Menyetujui pencairan dan mengunggah bukti transfer.💬 MODULE 6: CRM & Inbox SupportTerhubung dengan: contact-us.html, admin-inbox.html[ ] POST /api/support/ticket : (Publik) Mengirim pesan dari form Contact Us.[ ] GET /api/admin/support/tickets : (Admin) List pesan masuk.[ ] GET /api/admin/support/tickets/{id}/thread : (Admin) Membuka riwayat chat.[ ] POST /api/admin/support/tickets/{id}/reply : (Admin) Membalas pesan (Otomatis trigger email ke pelanggan).[ ] POST /api/admin/support/tickets/{id}/note : (Admin) Menambah Internal Note (tersembunyi dari pelanggan).⚙️ MODULE 7: Intelligence, CMS & System SettingsTerhubung dengan: admin-analytics.html, admin-settings.html, admin-blog-cms.html, admin-audit-logs.html[ ] GET /api/admin/analytics/dashboard : Mengambil data grafik (Revenue 6 bulan, Status Pesanan, Top Destinasi).[ ] GET /api/admin/audit-logs : Mengambil data jejak aktivitas (Read-only).[ ] GET /api/settings : Mengambil pengaturan global (Nama web, pajak %, API Key Gateway).[ ] PUT /api/admin/settings : Update pengaturan global.[ ] POST /api/upload/image : Mengunggah gambar (dari Text Editor / Cover) ke Cloud Storage (S3/Cloudinary) dan mengembalikan URL gambar.🔌 Kebutuhan Pihak Ketiga (Third-Party Integrations)Backend wajib menyiapkan akun dan API Keys dari layanan berikut:Payment Gateway: Midtrans / Xendit / Duitku (Untuk VA, QRIS, Credit Card).Cloud Storage: Amazon S3 / Google Cloud Storage / Cloudinary (Untuk menyimpan file gambar paket, bukti transfer, logo).WhatsApp API: Fonnte / Watzap / Twilio (Untuk mengirim OTP dan notifikasi tiket otomatis).Email Service (SMTP): SendGrid / Mailgun / Amazon SES (Untuk mengirim Invoice PDF dan Reset Password).
